Git Flow Description


1. Вступ

Цей документ надає опис основних принципів і процесів Git Flow та регламентує кроки для роботи з функціоналом, релізами та виправленнями багів.


2. Бранчі, які використовуються у Git Flow

Git Flow базується на наступних основних бранчах:

«master»: бранч, який містить стабільний код, готовий до випуску продуктивної версії.
«develop»: бранч, який є основним для розробки оновлень і нового функціоналу.
«feature»: бранчі функціоналу, які відгалужуються від «develop» для розробки конкретних функцій або вдосконалень.
«release»: бранчі релізу, які відгалужуються від «develop» для підготовки до випуску нової версії.
«hotfix»: бранчі виправлень, які відгалужуються від «master» для виправлення невеликих багів продуктивної версії.


3. Робота з Git Flow

3.1. Створення нового функціоналу

Виконайте команду «git checkout develop», щоб перейти на бранч «develop».
Виконайте команду «git pull origin develop», щоб отримати останні оновлення з «develop».
Виконайте команду «git checkout -b feature/my-feature», де «my-feature» - це ім'я вашого бранча.
Розробляйте новий функціонал на своєму бранчі.
Коли функціонал готовий, виконайте команду «git push origin feature/my-feature», щоб завантажити ваш бранч на сервер.

3.2. Підготовка до релізу

Виконайте команду «git checkout develop», щоб перейти на бранч «develop».
Виконайте команду «git pull origin develop», щоб отримати останні оновлення з «develop».
Виконайте команду «git checkout -b release/my-release», де my-release - це ім’я вашого бранча.
Виконайте необхідні тести, виправлення помилок та підготовку до випуску нової версії.
Коли реліз готовий, виконайте команду «git checkout master» та «git merge release/my-release», щоб обʼєднати реліз з бранчем «master».
Виконайте команду "git tag -a v1.0 -m «Версія 1.0»", щоб позначити реліз тегом.
Виконайте команду «git push origin master —tags", щоб завантажити зміни на сервер.

3.3. Виправлення багів

Виконайте команду «git checkout master», щоб перейти на бранч «master».
Виконайте команду «git pull origin master», щоб отримати останні оновлення з «master».
Виконайте команду «git checkout -b hotfix/my-hotfix», де «my-hotfix» - це ім’я вашого бранча.
Виправте баг на своєму бранчі.
Коли виправлення готове, виконайте команду «git checkout master» та «git merge hotfix/my-hotfix», щоб об’єднати виправлення з бранчем «master».
Виконайте команду "git tag -a v1.0.1 -m «Версія 1.0.1»", щоб позначити виправлення тегом.
Виконайте команду "git push origin master —tags", щоб завантажити зміни на сервер.
